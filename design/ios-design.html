iOS 設計書

SwiftUI ネイティブ iOS アプリケーション設計。バックエンド API は Web と共通で、ネイティブはバックエンドへ直接接続する（ADR-007）。本書は設計の正本とする。

フレームワーク SwiftUI
言語 Swift 5.10+
最小 iOS iOS 17
Xcode 16+
最終更新 2026-06-23
ステータス 実装進行中
関連: 要件・ ADR 007〜011・013（認証）・ Web フロントエンド設計書。 実装状況: 認証・Feed・Podcast 一覧・Settings（RSS 管理・アカウント管理）・管理者ユーザー管理は実装済み。音声再生（AVPlayer）は Task 6 未着手のため §8 は実装前の設計を据え置く。
バックエンド契約の要点（逸脱しないこと）: ①再生位置・既定設定（難易度／速度）はサーバー API ではなく端末ローカル（UserDefaults）保存（PATCH /podcasts/:id/position・PUT /settings は存在しない）。 ②RSS は /settings/sources（URL キー・ID なし）で操作。 ③audio_url は 1 時間失効の署名付き URL のため再生直前に再取得。 ④status は現行 Podcast レスポンスに未含有のため生成中ポーリングは Phase 2 課題（ADR-011）。 ⑤利用者はログイン必須（ADR-013）。セッショントークンは Keychain 保管、Authorization: Bearer で送出。

2. 画面構成・ナビゲーション

起動時は API 設定の有無、続いて認証状態（ADR-013）でルーティングする。設定済みなら refreshAuth()（GET /auth/me）でセッションを解決し、未ログインなら LoginView を挟んでから本体（ContentView）へ入る。本体は標準 TabView の 3 タブ（Feed / Podcast / Settings）。RSS 購読管理・アカウント管理・管理者によるユーザー管理は Settings タブ内に置く。

4. 認証・セッション（ADR-013）

利用者は username + パスワードでログインし、以降は不透明トークンで認証する。Web は httpOnly Cookie、iOS は Authorization: Bearer でトークンを送る（バックエンドの認証依存が両系統を吸収）。

認証モデル（Models/AuthModels.swift）

型内容
AuthUserusername / role（admin|user）/ displayName（display_name）。isAdmin 派生を持つ
LoginResponsetoken + user（POST /auth/login のレスポンス）
UserListResponseusers（GET /admin/users）
AuthStatus（enum）unknown（解決前）/ authenticated / unauthenticated

セッションストア（Networking/SessionStore.swift）

型役割
SessionStore（protocol）トークンの読み書き抽象
KeychainSessionStore本番。Keychain（generic password）に保管。service=com.newslisten.app.session・account=nl_session・アクセス属性 kSecAttrAccessibleAfterFirstUnlock（端末ロック連動・バックアップ外）
InMemorySessionStoreテスト用インメモリ実装

認証フロー

  • 起動解決: 設定済みなら AppState.refreshAuth() が GET /auth/me を呼ぶ。成功 → authStatus = .authenticated、失敗 → トークン破棄 + .unauthenticated。
  • ログイン: LoginView → POST /auth/login → completeLogin(response)（Keychain へトークン保存・currentUser 設定・トークン付き APIClient を再生成）。
  • ログアウト: logout() は POST /auth/logout をベストエフォートで呼び、必ず Keychain のトークンを破棄して .unauthenticated 化。
  • 失効への追従: 管理者による降格・PW リセット・削除でサーバー側セッションが失効すると、以降のリクエストは 401。これを未認証として扱い再ログインへ誘導する。

6. AppState 設計

AppState（ObservableObject・@MainActor）はアプリ全体のグローバル状態を保持する。API 接続設定・再生既定値は @AppStorage（UserDefaults）に、セッショントークンは SessionStore（Keychain）に永続化する。音声再生中の一時状態はメモリのみ。

プロパティ / メソッド保存先説明

defaultDifficulty / defaultPlaybackSpeedUserDefaults（@AppStorage）既定難易度（既定 toeic_900）・既定速度（既定 1.0）。端末ローカルのみ
articleOpenModeUserDefaults（@AppStorage）記事の開き方（in_app / external、既定 in_app）
authStatusメモリ（起動時に解決）unknown → authenticated / unauthenticated
currentUserメモリログイン中ユーザー（AuthUser?）

logout()—POST /auth/logout（ベストエフォート）+ 必ずトークン破棄 + .unauthenticated

isConfigured / apiClient（派生）計算で導出設定済み判定・トークン付き APIClient の生成

7. API クライアント設計

APIClient は @MainActor で動作し、ベース URL・API キー・セッショントークンを保持する。全リクエストに基盤の X-API-Key を付与し、ログイン後はさらに Authorization: Bearer <token> を付与する（ADR-013）。テスト可能性のため URLSessionProtocol を注入する。エラーは APIError 型（network / http(statusCode, detail) / decoding）。

メソッドパス用途
fetchFeedGET /feed記事フィード一覧
starArticle(id) / dismissArticle(id)POST /articles/{id}/star · /dismissStar / Dismiss
fetchPodcasts / fetchPodcast(id)GET /podcasts · /podcasts/{id}一覧 / 単体（再生直前の再取得）
fetchSources / addSource / deleteSourceGET/POST /settings/sources · DELETE ?url=RSS 一覧 / 追加 / 削除（URL キー）
login / logoutPOST /auth/login · /auth/logoutログイン / ログアウト
fetchMe / updateProfileGET · PATCH /auth/meユーザー取得 / 表示名更新
changePasswordPOST /auth/passwordパスワード変更
listUsers / createUser ADMINGET · POST /admin/usersユーザー一覧 / 作成
updateUser / deleteUser ADMINPATCH · DELETE /admin/users/{username}更新 / 削除

再生位置・既定設定の API は存在しないため、これらは端末ローカル（UserDefaults）のみで管理する。主要エラーコード: 401（キー不正・未認証/失効セッション）/ 403（admin 権限不足）/ 404（不存在）/ 409（重複・最後の admin 保護）/ 422（不正フィード）。

モックテスト戦略

URLSessionProtocol に準拠する MockURLSession を APIClient に注入し、固定レスポンスでデコード・エラー分岐を検証する（APIClient が @MainActor のためテストクラスも @MainActor 化）。セッション保管は InMemorySessionStore を注入してテストする。

8. 音声再生（AVPlayer）設計 TASK 6 未着手

AudioPlayerState（ObservableObject）は AVPlayer をラップし、再生対象エピソード・再生中フラグ・現在位置・総再生時間・再生速度を保持する。AppState 経由でアプリ全体に 1 つだけ共有する（Web の単一 <audio> 相当）。再生開始は次の手順。

  1. 既存の再生を停止・クリーンアップする。
  2. AVAudioSession を .playback カテゴリ・.spokenAudio モードで有効化し、バックグラウンド再生に対応する。
  3. 再生直前に GET /podcasts/{id} で再取得し、失効対策済みの署名付き URL で AVPlayerItem を生成。保存済み再生位置へ seek し、速度（rate）を設定して再生。
  4. 約 10 秒ごとに再生位置を UserDefaults（キー podcast_position:{id}）に保存する。サーバーへは送信しない。

ロック画面操作は MPNowPlayingInfoCenter / MPRemoteCommandCenter（play/pause/skipForward+30/skipBackward−15）、割り込みは AVAudioSession.interruptionNotification を購読し中断で一時停止・.shouldResume で再開する。

バックグラウンド再生設定

Info.plist の UIBackgroundModes に audio を追加。AVAudioSession.Category.playback + AVAudioSession.Mode.spokenAudio でポッドキャスト向けに設定する。
