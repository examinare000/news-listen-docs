iOS 設計書

SwiftUI ネイティブ iOS アプリケーション設計。Phase 2 対象（Web アプリが先行してリリースされた後に実装）。バックエンド API は Web と共通。

フレームワーク SwiftUI
言語 Swift 5.10+
最小 iOS iOS 17
Xcode 16+
フェーズ Phase 2
是正済み（2026-06-14）: 本ビジュアル案には実バックエンドとの契約乖離があったため、正本は 要件・ 設計（Markdown）・ ADR 007〜011 とする。 主な是正点: ①再生位置・既定設定（難易度/速度）はサーバー API ではなく端末ローカル（UserDefaults）保存（PATCH /podcasts/:id/position・PUT /settings は存在しない）。 ②RSS は /settings/sources（URL キー・ID なし）で操作（/subscriptions/:id は誤り）。 ③audio_url は 1 時間失効の署名付き URL のため再生直前に再取得。 ④status は現行 API レスポンスに未含有のため生成中ポーリングは Phase 2 課題。 本 HTML 内の該当記述は以下で是正済み。

5. AppState 設計

AppState はアプリ全体のグローバル状態（@Observable）を保持する。API 接続設定および再生既定値は @AppStorage 経由で UserDefaults に永続化し、音声再生中の一時状態はメモリのみで保持する。派生プロパティは保持値から計算で導出する。

プロパティ保存先説明
apiBaseURLUserDefaults 永続化（@AppStorage）API のベース URL（既定: 空）
apiKeyUserDefaults 永続化（@AppStorage）API キー（既定: 空）
defaultDifficultyUserDefaults 永続化（@AppStorage）既定の難易度（既定: toeic_900）。端末ローカルのみで管理
defaultPlaybackSpeedUserDefaults 永続化（@AppStorage）既定の再生速度（既定: 1.0）。端末ローカルのみで管理
currentPodcastメモリのみ現在再生中のエピソード
isPlayingメモリのみ再生中かどうか
currentTimeSecondsメモリのみ現在の再生位置（秒）
durationSecondsメモリのみ総再生時間（秒）
playbackSpeedメモリのみ現在の再生速度
isConfigured（派生）計算で導出apiBaseURL と apiKey の両方が設定済みなら true
apiClient（派生）計算で導出設定済みかつ URL が有効な場合に APIClient を生成して返す

6. API クライアント設計

APIClient は @MainActor で動作し、ベース URL・API キー・セッションを保持する。テスト可能性のため URLSessionProtocol を注入し、本番では URLSession を、テストではモックを差し替える。エラーは APIError 型で表現し、invalidURL（URL 不正）・httpError(statusCode, detail)（HTTP エラー）・decodingError（デコード失敗）の 3 種を持つ。

メソッド用途
フィード取得（fetchFeed）記事フィードの一覧を取得
記事の Star（starArticle(id)）指定記事に Star を付与
記事の Dismiss（dismissArticle(id)）指定記事を非表示にする
エピソード一覧取得（fetchPodcasts）ポッドキャストのエピソード一覧を取得
エピソード詳細取得（fetchPodcast(id)）エピソード詳細を取得。再生直前の再取得にも使用（署名付き URL 失効対策）
RSS ソース一覧取得（fetchSources）登録済み RSS ソースの一覧を取得
RSS ソース追加（addSource(name, url)）名前と URL を指定して RSS ソースを追加
RSS ソース削除（deleteSource(url)）URL をキーに RSS ソースを削除（DELETE /settings/sources?url=）

再生位置はサーバーに送らず端末ローカルに保存するため、再生位置 API は存在しない。既定難易度・既定再生速度の設定 API も存在せず、これらは端末ローカル（UserDefaults）のみで管理する。RSS ソースの実エンドポイントは /settings/sources で、URL をキーとし ID は持たない。

モックテスト戦略

URLSessionProtocol に準拠した MockURLSession をテスト用に定義し、APIClient に注入する。ViewModel のテストでは APIClient 自体をモック可能なプロトコル（APIClientProtocol）に差し替える。

7. 音声再生（AVPlayer）設計

AudioPlayerState（@Observable）は AVPlayer をラップし、再生対象エピソード・再生中フラグ・現在位置・総再生時間・再生速度を保持する。再生開始（play）は次の手順で行う。

  1. 既存の再生を停止・クリーンアップする。
  2. AVAudioSession を .playback カテゴリ・.spokenAudio モードで有効化し、バックグラウンド再生に対応する。
  3. 再生直前に GET /podcasts/{id} でエピソードを再取得し、失効対策済みの署名付き URL で AVPlayerItem を生成する。保存済み再生位置へ seek し、再生速度（rate）を設定して再生を開始する。
  4. 0.5 秒ごとに現在の再生位置と総再生時間（duration）を更新する。
  5. 約 10 秒ごとに再生位置を UserDefaults（キー: podcast_position:{id}）に保存する。サーバーへは送信しない。

その他の操作として、togglePlayPause は再生／一時停止を切り替え、seek は指定秒へ再生位置を移動し、setSpeed は再生速度を変更する（再生中のみ実際の rate へ反映）。クリーンアップ処理では位置保存タスクのキャンセル・時間オブザーバの解除・プレイヤーの停止と解放・再生中フラグの解除を行う。

バックグラウンド再生設定

Info.plist の UIBackgroundModes に audio を追加。AVAudioSession.Category.playback + AVAudioSession.Mode.spokenAudio でポッドキャスト向けの設定にする。
