1. 概要・技術スタック

項目技術理由
UI フレームワークSwiftUI (iOS 17+)宣言的 UI・モダン API・ライブアクティビティ等への拡張性

テストXCTest + Swift TestingViewModel の単体テスト・モック URLSession

SubscriptionsView Tab 3 — 購読管理（専用タブ）

RSS 購読ソースの一覧・追加・削除を行う専用画面。

要素詳細
ソース一覧List でソース名と URL を表示。デフォルト2件（HackerNews / Zenn）
スワイプ削除.onDelete → 確認アクションシート → DELETE /subscriptions/:id
追加ボタンツールバーの + ボタン → AddSubscriptionSheet をシート表示
AddSubscriptionSheet「ソース名」「RSS URL」TextField + 追加ボタン（インラインバリデーション）
バリデーションURL 形式チェック（クライアント）+ エラーレスポンスをインライン表示
エラー表示409 重複: 「この URL は登録済みです」、422: 「RSS フィードとして読み込めません」

AddSubscriptionSheet の詳細フロー

  1. 「ソース名」と「RSS URL」を TextField に入力
  2. 追加ボタンタップ（名前・URL が空の場合は無効）
  3. POST /subscriptions → 成功時: シートを閉じて一覧を更新
  4. 失敗時: シートを閉じず、TextField の下にエラーメッセージを表示

5. AppState 設計

// AppState.swift
@Observable
final class AppState {

  // MARK: - API 設定（UserDefaults 永続化）
  @AppStorage("api_base_url")   var apiBaseURL: String = ""
  @AppStorage("api_key")        var apiKey: String = ""
  @AppStorage("default_difficulty")    var defaultDifficulty: String = "toeic_900"
  @AppStorage("default_playback_speed") var defaultPlaybackSpeed: Double = 1.0

  // MARK: - 音声再生状態（メモリのみ）
  var currentPodcast: Podcast?
  var isPlaying: Bool = false
  var currentTimeSeconds: Double = 0
  var durationSeconds: Double = 0
  var playbackSpeed: Float = 1.0

  // MARK: - 派生プロパティ
  var isConfigured: Bool { !apiBaseURL.isEmpty && !apiKey.isEmpty }

  var apiClient: APIClient? {
    guard isConfigured, let url = URL(string: apiBaseURL) else { return nil }
    return APIClient(baseURL: url, apiKey: apiKey)
  }
}

6. API クライアント設計

// Networking/APIClient.swift
protocol URLSessionProtocol {
  func data(for request: URLRequest) async throws -> (Data, URLResponse)
}
extension URLSession: URLSessionProtocol {}

enum APIError: LocalizedError {
  case invalidURL
  case httpError(statusCode: Int, detail: String)
  case decodingError(Error)
}

@MainActor
final class APIClient {
  private let baseURL: URL
  private let apiKey: String
  private let session: URLSessionProtocol

  init(baseURL: URL, apiKey: String, session: URLSessionProtocol = URLSession.shared) {
    self.baseURL = baseURL; self.apiKey = apiKey; self.session = session
  }

  // Feed
  func fetchFeed() async throws -> FeedResponse
  func starArticle(id: String) async throws -> StarResponse
  func dismissArticle(id: String) async throws

  // Podcasts
  func fetchPodcasts() async throws -> PodcastListResponse
  func fetchPodcast(id: String) async throws -> Podcast
  func updatePosition(podcastId: String, seconds: Double) async throws

  // Subscriptions（RSS 管理）
  func fetchSubscriptions() async throws -> SubscriptionsResponse
  func addSubscription(name: String, url: String) async throws -> SubscriptionsResponse
  func deleteSubscription(id: String) async throws

  // Settings
  func fetchSettings() async throws -> Settings
  func updateSettings(_ settings: Settings) async throws -> Settings
}

モックテスト戦略

URLSessionProtocol に準拠した MockURLSession をテスト用に定義し、APIClient に注入する。ViewModel のテストでは APIClient 自体をモック可能なプロトコル（APIClientProtocol）に差し替える。

7. 音声再生（AVPlayer）設計

// Podcast/AudioPlayerState.swift
@Observable
final class AudioPlayerState {
  private var player: AVPlayer?
  private var timeObserver: Any?
  private var positionSaveTask: Task<Void, Never>?

  var currentPodcast: Podcast?
  var isPlaying: Bool = false
  var currentTime: Double = 0
  var duration: Double = 0
  var speed: Float = 1.0

  func play(podcast: Podcast) {
    // 1. 既存の再生を停止・クリーンアップ
    stopAndCleanup()

    // 2. AVAudioSession を .playback カテゴリに設定（バックグラウンド再生）
    try? AVAudioSession.sharedInstance().setCategory(.playback, mode: .spokenAudio)
    try? AVAudioSession.sharedInstance().setActive(true)

    // 3. AVPlayerItem 生成・再生位置復元
    guard let url = URL(string: podcast.audioUrl ?? "") else { return }
    let item = AVPlayerItem(url: url)
    player = AVPlayer(playerItem: item)
    player?.rate = speed
    player?.seek(to: CMTime(seconds: podcast.playbackPositionSeconds, preferredTimescale: 600))
    player?.play()

    // 4. 再生位置の定期更新（0.5 秒ごと）
    timeObserver = player?.addPeriodicTimeObserver(
      forInterval: CMTime(seconds: 0.5, preferredTimescale: 600),
      queue: .main
    ) { [weak self] time in
      self?.currentTime = time.seconds
      self?.duration = item.duration.seconds.isNaN ? 0 : item.duration.seconds
    }

    // 5. 10 秒ごとに PATCH /podcasts/:id/position を送信
    positionSaveTask = Task { ... }

    currentPodcast = podcast
    isPlaying = true
  }

  func togglePlayPause() { ... }
  func seek(to seconds: Double) { ... }
  func setSpeed(_ speed: Float) { player?.rate = isPlaying ? speed : 0; self.speed = speed }

  private func stopAndCleanup() {
    positionSaveTask?.cancel()
    if let obs = timeObserver { player?.removeTimeObserver(obs) }
    player?.pause()
    player = nil
    isPlaying = false
  }
}

バックグラウンド再生設定

Info.plist の UIBackgroundModes に audio を追加。AVAudioSession.Category.playback + AVAudioSession.Mode.spokenAudio でポッドキャスト向けの設定にする。

Phase 2 実装時の注意点

  • APNs Push 通知の実装（Podcast 生成完了通知）
  • オフライン再生のためのローカルキャッシュ（FileManager + Background Downloads）
  • OpenAI TTS への切り替えに伴う音声 URL 形式の変更対応
