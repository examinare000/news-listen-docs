# docs/plan — 実装計画フォルダの運用

このフォルダは**実装計画の整理・タスク分解・実行（エージェントへの指示書）専用**の作業領域。

## ライフサイクル

1. **作成**: 機能実装の着手時に計画書・タスク指示書を置く（例: `<日付>-<対象>.md` または `<日付>-<対象>/` ディレクトリ）
2. **実行中**: Living Documentation として進捗チェックボックス・決定事項を随時更新する
3. **完了後**: 計画書は**削除**し、内容を恒久ドキュメントへ移す
   - 確定したアーキテクチャ決定 → `docs/adr/`（agent-rules/30 のテンプレートに従う）
   - 確定した仕様・画面挙動 → `docs/spec/`
   - 機能的な残課題（Phase 2 候補） → `docs/prd/` の該当セクション
   - 技術的負債 → `docs/tech/`
   - 運用手順・開発ワークフローの学び → `docs/operations/`

過去の計画書の経緯は git 履歴で参照する（削除済み: `2026-06-10-web-frontend.md`、`2026-06-12-webui-restyle/` ほか）。
