1. システム概要

news-listen は HackerNews・Zenn.dev 等の海外テックニュースを毎日自動収集し、AI が生成した Podcast 形式の音声コンテンツとして配信するパーソナル向けアプリケーションです。

システムコンテキスト図

主要フロー

定時バッチ（毎日 06:00 JST）

  • Cloud Scheduler が3ジョブを順次起動
  • RSS フェッチ → 本文抽出 → Firestore 保存
  • Gemini でユーザー関心スコアを再計算
  • Star 済み記事の Podcast を生成・保存
  • 06:30 JST までに完了

オンデマンド Podcast 生成（Star 操作時）

  • iOS/Web → API へ POST /articles/:id/star
  • API が Cloud Tasks へタスク投入（202 即返却）
  • バックグラウンドで Gemini スクリプト生成
  • Gemini TTS で音声生成 → Storage 保存
  • Firestore ステータス更新（完了通知）
  • 所要時間：Star から約 1〜2分

3. 主要設計決定事項

Web 先行・iOS 後追
バックエンド API を共通化し、まず Next.js Web アプリで機能を検証。iOS アプリは同一 API を使って後から実装する。
TTS: MVP は Gemini TTS、Phase 2 で OpenAI TTS に移行
MVP では google-genai の TTS で実装をシンプルに保つ。Phase 2 以降、話者 A/B 並列生成（掛け合い形式）が必要になった段階で OpenAI TTS に切り替える。
オンデマンド Podcast 生成に Cloud Tasks を使用
Star 操作から生成完了まで 1〜2 分かかるため、Cloud Tasks で非同期化。API は 202 Accepted を即返し、タイムアウトリスクを回避する。
マルチユーザー化：独自セッション認証で解決済み（ADR-013）
当初は固定値 default で個人利用を想定していたが、解決済み。Firebase Auth は見送り、既存スタック内の独自セッション認証でマルチユーザー化した。ログインユーザーの user_id（users コレクション）で識別し、username + パスワードでログイン、admin / user ロールを持つ。パスワードは bcrypt でハッシュ化。トークンは不透明値とし、サーバ側 Session（Firestore）で管理して即時失効可能。生トークンは保存せず SHA-256 ハッシュで保持する。Web は httpOnly Cookie nl_session、iOS は Authorization: Bearer + Keychain で受け渡す。設計は ADR-013。
RSS 管理は専用画面として独立
購読サイト一覧・URL 追加・削除は Settings とは別の独立した画面（Web: /subscriptions、iOS: SubscriptionsView）に配置し、視認性を高める。
API キー（基盤）＋セッション認証（利用者）の二層
基盤の共有 X-API-Key ヘッダーは据え置き（Secret Manager で管理）。その上で、利用者は username + パスワードによるセッション認証（ADR-013）でログインし、各リクエストは Web が httpOnly Cookie nl_session、iOS が Authorization: Bearer でセッショントークンを提示する。
admin 限定のユーザー管理とセッション安全性（ADR-013）
admin ロールのみがユーザーの作成・ロール変更・パスワードリセット・削除を行える。最後の admin を保護（降格・削除は 409 で拒否）し、自己ロックアウトも防止する。降格・パスワードリセット・削除の各操作では対象ユーザーの既存セッションを失効させる。

4. スコープ

🟣 MVP（P0）

  • Feed タブ（記事一覧・スワイプ/ボタン操作）
  • パーソナライズレコメンド
  • RSS 購読管理専用画面
  • Podcast タブ（一覧・再生）
  • オンデマンド生成（Star後1-2分）
  • 日本語イントロ + 英語本編
  • 難易度 6 段階・速度 8 段階
  • 再生位置保存
  • Settings（難易度・速度デフォルト）
  • Cloud Tasks 非同期生成
  • Web アプリ（Next.js）先行リリース
  • セッション認証・マルチユーザー管理 ✓ 実装済み（ADR-013）

🟠 Phase 2（P1）

  • iOS アプリ（SwiftUI）
  • 日次ダイジェスト自動生成
  • OpenAI TTS への移行（話者 A/B）
  • オフライン再生
  • Push 通知（生成完了）
  • マルチユーザー化 ✓ 実装済み（独自セッション方式・ADR-013）
  • ストレージ管理画面

⚪ Phase 3（P2）

  • Apple Watch 対応
  • 記事全文検索
  • Web Push 通知
  • ブラウザ拡張（Web クリップ）
  • PWA 化
