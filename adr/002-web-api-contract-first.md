# ADR-002: Web UI は実装済み API 契約を正とし、未対応機能の UI を実装しない

**ステータス:** 採用済み
**決定日:** 2026-06-10（2026-06-12 の WebUI リスタイルで再確認・拡張）
**対象読者:** Web フロントエンド開発者・デザイナー

## 背景

デザインモック（`docs/design/web-design.html`・`docs/design/app-ui.html`）はバックエンド確定前に作成されており、実 API（`backend/api/`）に存在しないエンドポイント・フィールドを前提とした UI を多数含む。モックを正として実装すると、動かない UI や嘘の表示（実際には起きない「生成開始」等）が生まれる。

## 検討した選択肢

| 選択肢 | メリット | デメリット |
|---|---|---|
| モックを正とし、必要な API をバックエンドに追加 | デザイン再現度が最大 | バックエンド変更が Web 実装をブロック。1 ブランチ = 1 目的に反する |
| **実 API 契約を正とし、未対応 UI は実装しない（差分は決定表で明文化）** | Web 単独で完結。嘘の表示が生まれない | デザインモックとの乖離が残る |

## 決定

実 API 契約（`backend/api/` の実コード）を唯一の正とし、API が提供しない機能の UI は実装しない。デザインとの乖離は計画時に決定表として明文化し、確定後は設計・仕様の正本（`docs/design/web-design.html`）へ反映する。

主な適用結果:

| モックの UI | 決定 | 根拠 |
|---|---|---|
| Podcast 生成ステータス（生成中/完了/失敗）バッジ・ポーリング・「新規生成」ボタン | 実装しない。代替として手動リフレッシュボタン | `PodcastResponse` に `status` がなく、生成は日次バッチのみ |
| 再生位置のサーバー同期 | localStorage（`podcast_position:{id}`）のみ | `PATCH /podcasts/:id/position` が存在しない |
| デフォルト難易度の設定 UI | 実装しない（説明文を表示） | 難易度はバッチ側環境変数で決まり API から変更不可 |
| 購読ソースの有効/無効トグル・行 ID | 実装しない | ソースは `{name, url}` のみで `id`・`enabled` が存在しない |
| フィードの「未読」タブ・既読件数バッジ | 実装しない（タブは「すべて/★スター済み」のクライアントフィルタのみ） | 既読管理 API がない |
| Star 時の「生成を開始しました」トースト | 「Star しました。Podcast は次回バッチ実行時に生成されます」 | 生成は即時開始されない |
| サイドバーの「バッチ: 正常稼働中」表示 | 実装しない | 監視 API がなく、嘘の表示は置かない |

## 理由

個人利用の MVP において、UI のための API 拡張よりも「動くものを正直に見せる」ことを優先。バックエンドと Web の変更を疎結合に保ち、各ブランチの目的を単一にするため。

## 結果（影響）

- 仕様の正は常に `backend/api/` 実コード > spec > PRD > デザインモックの順（spec ヘッダーに明記）
- API 拡張（status 公開・position API・settings API・CORS 追加）が入った場合は、新 ADR を起こした上で対応 UI を解禁する（Phase 2 候補は PRD §11 参照）
