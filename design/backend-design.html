4. API エンドポイント仕様

全エンドポイントに X-API-Key: {key} ヘッダーが必要。/health のみ認証不要。

Feed

GET /feed

当日のレコメンド済み記事一覧を返す。

レスポンス型説明
articlesArticle[]スコア降順（最大 50 件）
datestringYYYY-MM-DD

Articles

POST /articles/{id}/star 非同期

記事を Star 登録し、Podcast 生成キューに投入する。

レスポンス型説明
status"starred"
article_idstring
podcast_idstring生成中の Podcast ID

POST /articles/{id}/dismiss

記事を Dismiss（以降フィードに表示しない）。

Podcasts

GET /podcasts

ユーザーの Podcast エピソード一覧（最新 50 件）。

GET /podcasts/{id}

エピソード詳細。status で生成中/完了/失敗を確認できる。

PATCH /podcasts/{id}/position

再生位置を保存する。

リクエスト型説明
position_secondsnumber現在の再生位置（秒）

RSS 購読管理（Subscriptions）

GET /subscriptions

購読中の RSS ソース一覧を返す。

{
  "sources": [
    { "id": "hnrss_frontpage", "name": "HackerNews", "url": "https://hnrss.org/frontpage", "enabled": true },
    { "id": "zenn_feed",       "name": "Zenn.dev",   "url": "https://zenn.dev/feed",       "enabled": true }
  ]
}
// id: SHA-256(url)[:12] で自動生成（POST /subscriptions 時にサーバー側で付与）

POST /subscriptions

新しい RSS ソースを追加する。

リクエスト型バリデーション
namestring1〜50 文字
urlstringHTTP/HTTPS URL・重複不可
エラー説明
400URL フォーマット不正
409URL 重複
422RSS フィードとして取得不可（バリデーション）

DELETE /subscriptions/{id}

RSS ソースを削除する。デフォルトソース（HackerNews / Zenn）は削除可能（確認なし）。

Settings

GET /settings

ユーザー設定を返す（難易度・再生速度デフォルト値）。

POST /settings

ユーザー設定を更新する。

フィールド型説明
default_difficultystringtoeic_600 / toeic_900 / ielts_55 / ielts_7 / eiken_2 / eiken_p1
default_playback_speednumber0.5〜2.5

内部エンドポイント（Cloud Tasks からのみ）

POST /internal/generate-podcast

Cloud Tasks ワーカー用。X-CloudTasks-QueueName ヘッダーで呼び出し元を検証する。

5. Firestore データモデル

articles コレクション

// ドキュメント ID: SHA-256(url)[:20]
{
  "title":             string,
  "url":               string,
  "source":            string,       // "HackerNews" | "Zenn.dev" | ユーザー定義名
  "content":           string,       // フェッチ済み本文（最大 10,000 文字）
  "published_at":      timestamp,
  "fetched_at":        timestamp,
  "content_fetched_at": timestamp | null
}

userPrefs コレクション

// ドキュメント ID: user_id（MVP = "default"）
{
  "user_id":                string,
  "starred_article_ids":    string[],
  "dismissed_article_ids":  string[],
  "rss_sources": [
    { "id": string, "name": string, "url": string, "enabled": boolean }
  ],
  "default_difficulty":     string,   // Difficulty enum
  "default_playback_speed": number,   // 0.5〜2.5
  "digest_enabled":         boolean,  // P1
  "digest_article_count":   number    // 3〜10、P1
}

recommendations コレクション

// ドキュメント ID: {user_id}_{YYYY-MM-DD}
{
  "user_id":    string,
  "date":       string,     // "2026-05-31"
  "ranked_articles": [
    { "article_id": string, "score": number }  // score: 0.0〜1.0
  ],
  "generated_at": timestamp
}

podcasts コレクション

// ドキュメント ID: UUID v4
{
  "type":                   "single" | "digest",
  "article_ids":            string[],
  "difficulty":             string,
  "audio_url":              string | null,       // 完了後に設定
  "japanese_intro_text":    string,
  "duration_seconds":       number,
  "status":                 "processing" | "completed" | "failed" | "partial_failed",
  "error_message":          string | null,
  "playback_position_seconds": number,           // 再生位置
  "created_at":             timestamp,
  "completed_at":           timestamp | null,
  "user_id":                string
}

Firestore インデックス

コレクションフィールド（複合インデックス）用途
articlespublished_at DESC最新記事一覧取得
podcastsuser_id ASC, created_at DESCユーザー別エピソード一覧
podcastsuser_id + article_ids (array_contains) + difficulty + typeStar 済み記事の重複生成防止

6. Cloud Tasks 設定

項目設定値理由
キュー名podcast-generation
最大同時実行数5Gemini TTS のレートリミット対策
タスクタイムアウト10 分スクリプト生成 + TTS 生成の合計時間を考慮
最大リトライ数3一時的な Gemini API エラーを吸収
リトライ間隔指数バックオフ（初回 30s）
ターゲット URLhttps://{api-url}/internal/generate-podcast
認証Cloud Tasks → OIDC トークン（Cloud Run が検証）

7. エラーハンドリング方針

シナリオ対応
RSS フェッチ失敗ソース単位でスキップ、ログ記録。他のソースは継続処理。
本文スクレイピング失敗RSS の summary をそのまま使用（空でも続行）。
Gemini スクリプト生成失敗Cloud Tasks がリトライ（最大 3 回）。全リトライ後 status=failed。
Gemini TTS 失敗イントロ or 本編の片方が失敗した場合は status=partial_failed、音声は空セグメントをスキップして結合。
Cloud Storage 保存失敗リトライ（最大 3 回）。失敗時 status=failed。
Gemini レコメンド失敗全記事にデフォルトスコア 0.5 を設定してフィードを返す。
